# Opal 2.13 Released

We are pleased to announce that Opal 2.13 is now available. [Opal](http://www.obiba.org/pages/products/opal/) is OBiBa’s core database application
    for biobanks.

Thanks to the support of the [Canarie ](https://www.canarie.ca/) program, this release extends the
  [datasource plugin service interface ](https://github.com/obiba/opal/tree/master/opal-spi/src/main/java/org/obiba/opal/spi/datasource)
  with plugins using the R server to import or export data:

    - from/to a delimited format file ([ReadR ](https://github.com/obiba/opal-datasource-readr)
      based on [readr ](https://www.rdocumentation.org/packages/readr)),

    - from/to a Excel file ([ReadXL ](https://github.com/obiba/opal-datasource-readxl)
      based on [readxl ](https://www.rdocumentation.org/packages/readxl)
      and on [writexl ](https://www.rdocumentation.org/packages/writexl)),

    - from a Google Sheet ([GoogleSheets ](https://github.com/obiba/opal-datasource-googlesheets4)
      based on [googlesheets4 ](https://googlesheets4.tidyverse.org/)).

A new [analysis plugin service interface ](https://github.com/obiba/opal/tree/master/opal-spi/src/main/java/org/obiba/opal/spi/analysis)
  has also been added to integrate in the user interface data/meta-data analysis routines. The analyses are designed like a per-table test suite with a output report.
  As a proof of concept the plugin [Validate ](https://github.com/obiba/opal-analysis-validate)
  based on the R package [validate ](https://www.rdocumentation.org/packages/validate) has been developed.

Plugins are available from our [ plugin repository ](https://plugins.obiba.org/).
  Please contact us if you want to develop and refer a new plugin. Using R packages to import/export and analyse data opens a lot of possibilities!

See [Opal documentation ](http://opaldoc.obiba.org) for installation and operation instructions.
