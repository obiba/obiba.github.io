# Mica2 5.0 Released

We are pleased to announce that Mica2 5.0 is now available. [Mica](/pages/products/mica/) is OBiBa’s data web portal for epidemiological studies and harmonization initiatives.

To account for the new standards proposed by [Maelstrom Research ](https://www.maelstrom-research.org) to document harmonization initiatives and outputs, this release includes the following modifications:

- The entity of Harmonization Initiative replaces Harmonization Study.
- The entity of Harmonization Protocol replaces Harmonized Dataset.
- New fields in both entities are added to better describe the harmonization process.
- The Population and Harmonization Design fields are removed from the Harmonization Study (now Harmonization Initiative).
- Two distinct search pages are created specifically for: Individual Studies and Harmonization Initiatives.

Please review [Mica 5.0 Upgrade Notes ](https://github.com/obiba/mica2/wiki/Mica-5.0-Upgrade-Notes) before upgrading your current installation of Mica.

This release also includes the following features:

- Data Access Requests can be archived regardless of their status.
- Data Access Requests archive operations are logged as an action and therefore recorded in the history.
- Notifications are disabled when a Data Access Request is archived.
- Two-factor authentication can be enabled for enhanced security (also see [Agate 2.6](/pages/news/release/agate/2.6/)).
- Login is not required to use the cart (configurable).

This release was possible thanks to the funding of the [EUCAN Connect ](https://www.eucanconnect.eu/) ([Coral](https://www.eucanconnect.eu/coral-docker-stack-deployment/) integration) program, and to the contribution of [CARTaGENE ](https://www.cartagene.qc.ca/) and [Maelstrom Research ](https://www.maelstrom-research.org/) (including development contributions).

See [Mica documentation ](http://micadoc.obiba.org) for installation and operation instructions. Note that a [Docker Image Installation ](http://micadoc.obiba.org/en/latest/admin/installation.html#docker-image-installation) section has been added.
