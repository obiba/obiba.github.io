# Mica2 3.6 Released

We are pleased to announce that Mica2 3.6 is now available. [Mica](/pages/products/mica/) is OBiBa’s data web portal for epidemiological studies and networks.

This release introduces the following:

- **Individual Study Model**, the Access section has been extended to capture more details regarding data sharing and dissemination.
- **Opal Views Download**, this feature permits Mica users to generated and archive importable Opal Views based on variable lists.

Only users wishing to update their Individual Study model to have the new Access section must follow the following data migration steps:

- Stop Mica database.
- **Backup** Mica database.
- Go to Administration > Individual Study Configuration and update your current schema and definition using this [schema](https://github.com/obiba/mica2/blob/master/mica-core/src/main/resources/config/study-form/collection-schema.json) and [definition](https://github.com/obiba/mica2/blob/master/mica-core/src/main/resources/config/study-form/collection-definition.json) and save the changes.
- Unzip [access-section-update-migration.zip](https://github.com/obiba/mica2/releases/download/3.6.0/access-section-update-migration.zip) and run the migration script ./access_migration.sh.
- Restart Mica server
- Go to Administration > Individual Study Configuration > Search (TAB) and save so the Individual Study taxonomies get re-indexed.
- Go to Administration > Indexing and index Studies.

See [Mica documentation ](http://micadoc.obiba.org) for installation and operation instructions.
