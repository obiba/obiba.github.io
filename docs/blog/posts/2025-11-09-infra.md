# Infrastructure Migration

We have migrated our infrastructure from self-hosted servers to software-as-a-service (saas) open source software offers:

- Source code, issue management and release packages are in [GitHub ](https://github.com/obiba/),
- Continuous integration is performed by [Travis-ci ](https://travis-ci.org/obiba),
- Snapshot maven artifacts are in [JFrog Artifactory repositories ](https://obiba.jfrog.io),
- Released maven artifacts and debian/rpm packages are in [JFrog Bintray repositories ](https://bintray.com/obiba/maven),
- Documentations are hosted by [Read the Docs ](https://readthedocs.org/).

As a consequence **system administrators must update Debian and/or RPM package repositories references**: see the [Debian repository](/pages/pkg) and [RPM repository](/pages/rpm) instructions.

See also the [Infrastructure description](/pages/infra) for more details.
