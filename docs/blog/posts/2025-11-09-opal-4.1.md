# Opal 4.1 Released

We are pleased to announce that Opal 4.1 is now available. [Opal](https://www.obiba.org/pages/products/opal/) is OBiBa’s core data management application for biobanks.

This release of Opal introduces the following new features:

- SQL API, to perform SQL queries on Opal's tables from the [Project's web interface ](https://opaldoc.obiba.org/en/latest/web-user-guide/projects/tables.html#sql), or using the [opalr R package ](https://rdocumentation.org/packages/opalr/versions/2.1.0/topics/opal.sql) or from the [Opal SQL Python command ](https://opaldoc.obiba.org/en/latest/python-user-guide/datasources/sql.html),
- new [Personal Access Token read-only scope ](https://opaldoc.obiba.org/en/latest/web-user-guide/my-profile.html#personal-access-tokens), with or without access to individual-level data,
- data assignment to the DataSHIELD/R server are much faster thanks to a new caching system.

No breaking changes are expected with this release.

As a reminder, following the announcement that the Bintray repository will be deprecated, the OBiBa software are now available at [obiba.jfrog.io](https://obiba.jfrog.io/). What you need to do is to follow instructions for the [Debian repository](https://www.obiba.org/pages/pkg/) and for the [RPM repository](https://www.obiba.org/pages/rpm/).

This release was possible thanks to the funding of the [EUCAN-Connect ](https://www.eucanconnect.eu/) program.

See [Opal documentation ](http://opaldoc.obiba.org) for installation and operation instructions. We recommend to have a look at the section about [Docker Image Installation ](http://opaldoc.obiba.org/en/latest/admin/installation.html#docker-image-installation).
