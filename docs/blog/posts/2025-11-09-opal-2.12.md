# Opal 2.12 Released

We are pleased to announce that Opal 2.12 is now available. [Opal](http://www.obiba.org/pages/products/opal/) is OBiBa’s core database application for biobanks.

Thanks to the support of the [Canarie ](https://www.canarie.ca/) program, this release introduces a [datasource plugin service interface ](https://github.com/obiba/opal/tree/master/opal-spi/src/main/java/org/obiba/opal/spi/datasource) that allows to import data:

- from a file (for instance [SPSS ](https://github.com/obiba/opal-datasource-spss)),
- from a database (for instance [Limesurvey ](https://github.com/obiba/opal-datasource-limesurvey)),
- from an application web services (for instance [REDCap ](https://github.com/obiba/opal-datasource-redcap)).

Plugins are available from our [ plugin repository ](https://plugins.obiba.org/). Please contact us if you want to develop and refer a new plugin.

Other notable features are:

- the identifier mapping (at import and export) applies to variable values declared as being entity identifiers (referenced entity type property),
- SPSS missing discrete values and ranges are now mapped to missing categories, both at import and export,
- system log files can be downloaded from the administration pages,
- users with "Add table" permission can import data,
- SSL connection with MongoDB is now supported,
- user ban policy has been added (temporary ban after several login failures).

See [Opal documentation ](http://opaldoc.obiba.org) for installation and operation instructions.
