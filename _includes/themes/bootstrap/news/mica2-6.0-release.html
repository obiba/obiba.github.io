We are pleased to announce that Mica2 6.0 is now available. Mica is OBiBa's data web portal for epidemiological studies and harmonization initiatives.

This Mica 6.0 release requires Java 21.

Upgrade actions:

  • Java runtime: If the default Java installed in your system is not Java 21, you must install Java 21 (any vendor), update the path to the java executable in the file /etc/default/mica2, and restart the mica2 service.
  • Plugins:
    • Mica Search Plugin: The latest Mica search plugin is downloaded on the first run of Mica 6.0. Update settings like clusterName and transportAddresses in site.properties.
    • Mica Tables SPSS and Mica Taxonomies Files: Unzip the latest versions as directories under MICA_HOME/plugins and update their site.properties settings accordingly.

Other new features:

  • Added configurable roles mapping from groups.
  • Support for no search engine when using Mica solely for data access workflows.
  • Addition of new permissions for different levels of variable summary visibility.

Additional notes:

  • Mica 6.0 requires Elasticsearch 8.16.1, older versions are no longer supported.
  • No upgrade actions required when Mica is deployed with Docker, just pull the latest image.
  • The Mica API has NOT been modified: it will seamlessly integrate with MicaR and Mica Python Client.
  • Mica 6.0 is compatible with older versions of Opal and Agate.

Mica 6.0 Search Engine

By default, Mica 6.0 requires an Elasticsearch 8.16.1 server unless it is configured to manage Data Access Requests only.

Refer to Install Elasticsearch with Docker for the recommended installation instructions or Elastic's installation guide for all types of server installation instructions.
