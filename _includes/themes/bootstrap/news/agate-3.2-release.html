Since Agate 3.0, this release requires Java 21. If you are still using Java 8, you must upgrade to Agate 3.0 first.

New features and changes:

  • When a user registers from an external realm (most likely using OpenID Connect), through an application, the groups that are assigned to this new user can be configured per application.
  • The application configuration file is now application-prod.yml. Migration from application.yml to application-prod.yml is done automatically.
  • The user name can now be an email address.
