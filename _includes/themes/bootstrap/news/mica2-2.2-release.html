We are pleased to announce that Mica2 2.2 is now available. Mica is OBiBaâ€™s data web portal for epidemiological studies and networks.

As with version 2.0, this release offers the possibility to fully configure the data model of the documents (network, study, datasets) that are published in the data portal.

This release contains the following features and improvements:

  • possibility of hiding search criteria,
  • better ordering of search results: datasets are ordered chronologically and variables are ordered by their corresponding study, population and data collection dates as well as their order in the source questionnaire,
  • better ordering of documents listing: sorting networks by the number of studies and sorting study datasets chronologically.
Upgrade with authentication on MongoDB. The connection settings to MongoDB have been changed for a standard connection string since Mica2 2.0. Administrators having defined a user/password to connect to MongoDB (this is recommended) should read the MongoDB configuration documentation before upgrading the Mica server.

Make sure to backup your installation as Mica2 server upgrade from 1.5 will affect the data: the server will automatically upgrade the data model and re-index the published documents. Default configuration is the data model specified and used by Maelstrom Research . For the Mica Drupal modules and theme, see upgrade instructions .

See also Mica documentation for installation and operation instructions.
